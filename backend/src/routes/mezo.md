# mezo reference

**المسار:** `backend/src/routes`

## وظيفة هذا المكان
كل API endpoints الفعلية. كل ملف هنا يمثل مجموعة endpoints تخص جزءًا معينًا من النظام.

## ملاحظة مهمة
إذا أضفت endpoint جديد هنا، اربطه أيضًا في `backend/src/index.ts` ثم أضف استدعاءه من `frontend/lib/api.ts` أو `frontend/lib/admin-api.ts` حسب الاستخدام.

## المحتويات
- **ملف `banners.ts`**: API لإدارة البانرات والعروض من الأدمن وقراءتها للواجهة.
- **ملف `cart.ts`**: API السلة: جلب عناصر الكارت، الإضافة، التعديل، والحذف.
- **ملف `categories.ts`**: API الأقسام/التصنيفات مع CRUD وإظهار الأقسام للواجهة.
- **ملف `chat.ts`**: API الشات أو المساعد داخل الموقع.
- **ملف `contact.ts`**: API استقبال رسائل تواصل العملاء وإدارتها من الأدمن.
- **ملف `faq.ts`**: API الأسئلة الشائعة.
- **ملف `orders.ts`**: API الطلبات. ينشئ الطلبات بشكل آمن من أسعار القاعدة ويتابع الحالات.
- **ملف `products.ts`**: API المنتجات: عرض، بحث، فلترة، وإنشاء/تعديل من الأدمن مع variants.
- **ملف `reviews.ts`**: API المراجعات: إضافة التقييمات وجلب المعتمد منها أو اعتماده.
- **ملف `settings.ts`**: API إعدادات الموقع العامة المخزنة كمفاتيح وقيم.
- **ملف `social.ts`**: API إدارة روابط السوشيال والتواصل.
- **ملف `upload.ts`**: API رفع الصور/الملفات للمنتجات أو البانرات مع حفظها في uploads.
- **ملف `users.ts`**: API المستخدمين والتوثيق: register/login/me/list/roles.

## كيف أتعامل معه لاحقًا
استخدم هذا الملف كمرجع سريع قبل التعديل. لو ستغير منطقًا هنا، راجع الملفات المرتبطة المذكورة أعلاه حتى لا يحدث كسر في جزء آخر من المشروع.
