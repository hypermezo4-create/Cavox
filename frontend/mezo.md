# mezo reference

**المسار:** `frontend`

## وظيفة هذا المكان
واجهة Next.js. تحتوي صفحات المتجر، صفحات الأدمن، المكونات، وطبقة الربط مع الباك إند.

## المحتويات
- **مجلد `app`**: مسارات App Router في Next.js. هنا صفحات المتجر، auth، الأدمن، وبعض API routes الخاصة بالفرونت.
- **مجلد `components`**: كل المكونات المشتركة للواجهة، مقسمة حسب المجال: admin، brand، chat، store، layout، وغيرها.
- **مجلد `lib`**: طبقة utilities والـ API clients والربط مع الباك إند.
- **مجلد `styles`**: الأنماط العامة للمشروع.
- **مجلد `types`**: تعريفات TypeScript الإضافية أو توسيعات مكتبات مثل NextAuth.
- **ملف `.env.example`**: متغيرات البيئة المطلوبة للفرونت مثل رابط الـ backend وNextAuth.
- **ملف `middleware.ts`**: Middleware خاص بـ Next.js لحماية مسارات الأدمن أو auth أو إعادة التوجيه.
- **ملف `next-env.d.ts`**: ملف TypeScript المولد من Next.js لدعم الأنواع الأساسية.
- **ملف `next.config.js`**: إعدادات Next.js للواجهة.
- **ملف `package.json`**: اعتماديات وسكربتات الواجهة الأمامية.
- **ملف `postcss.config.js`**: إعدادات PostCSS/Tailwind.
- **ملف `tailwind.config.ts`**: إعدادات Tailwind للواجهة.
- **ملف `tsconfig.json`**: إعدادات TypeScript للفرونت.

## كيف أتعامل معه لاحقًا
استخدم هذا الملف كمرجع سريع قبل التعديل. لو ستغير منطقًا هنا، راجع الملفات المرتبطة المذكورة أعلاه حتى لا يحدث كسر في جزء آخر من المشروع.
